Technical Explanations

Evaluation

The engine will use a neural network based evaluation. The evaluation process goes as follows:

  • pass each timeline through a LSTM
  • combine the results and the final board's position, creating three tensors: minimum, maximum and average
  • use a FNN to compute 3 values: the probability of a win, a draw and a loss

Search

The engine will have 2 search modes: normal search and mate search.

In normal search, the engine will use minimax with alpha-beta pruning, move ordering and mate distance pruning, as well as staged move generation. It will for example generate easy softmate scenarios first, and so if any of them are checkmate or above beta, it will instantly cut off.
Note that 5D chess has a very high branching factor that grows exponentially with the number of timelines, and so speed and selectivity are key here.
This is why the old evaluations will be conserved: the advantage of using LSTMs is that whenever you make a move, the LSTM doesn't need to be recomputed entirely, but just for that last board, and when you take back that move, you can just go back to the older results.

In mate-only search, the engine will use minimax with the same backward pruning as the normal search, but with the addition of proof-number search, to speed up the checkmate search even further. This algorithm will also be selective if you enable the selectivity flag, so even if you are searching with a depth N, maybe some moves won't get seen.

The checkmate detection algorithms will be the following for both searches, except if better ones are found:

  • Shad's optimized search for fewer than 4 timelines
  • tesseract's hypercuboid algorithm for 4 or more timelines
